## 📛 建议文件名

建议将 `LUKS.py` 改名为 **`GhostFile.py`** ——  
“Ghost” 意为“幽灵”，既体现文件的伪装（表面无害），又暗示隐写的“不可见”特性，简洁且不与其他安全工具重名。

## 📝 README.md

```markdown
# GhostFile 🔏👻

**多层混合加密 + 零宽字符伪装**  
一个 Python 命令行工具，将任意文件深度加密，并可将其伪装成一段无意义的纯文本——用任何编辑器打开都只能看到你指定的“表面文字”，真实密文通过不可见的零宽字符隐藏其中。

---

## ✨ 特性

- **多层嵌套加密**  
  `zlib 压缩 → 凯撒字节位移 → AES-256-CBC → Base64 → 零宽字符编码`
- **伪装模式（隐写）**  
  加密后的文件在记事本中**仅显示你指定的文字**（如“糯米狐”），真实密文不可见。
- **自适应伪装源**  
  `-w` 参数可直接跟文字，也可跟一个文本文件路径（读取其内容作为伪装）。
- **全格式支持**  
  输入可以是 `.md` `.pdf` `.docx` `.png` `.mp4` 等任意文件类型，完全二进制安全。
- **跨平台不可见**  
  零宽字符在 Windows / Linux / macOS / Android / iOS 所有主流系统的文本编辑器中均不显示。
- **压缩减容**  
  内置 zlib 压缩原始数据，有效降低零宽编码后的文件体积（膨胀比从约 32 倍降至约 12 倍）。
- **密码派生密钥**  
  使用 PBKDF2-HMAC-SHA256 高强度密钥派生，抗暴力破解。

---

## ⚖️ 优缺点

### ✅ 优点
- **极高隐蔽性**：表面是完全无害的文本，容易被忽略。
- **强加密保障**：AES-256-CBC + 凯撒双混淆，即使提取出密文也难以破解。
- **通用兼容**：纯 Python 实现，不依赖操作系统特性，加解密完全跨平台。
- **灵活伪装**：可随意定制表面文字，或直接复制一篇长文章作为伪装载体。
- **原文件零修改**：加密后产生新文件，原始文件不受影响。

### ❌ 缺点
- **文件膨胀**：零宽编码会导致体积显著增大（约为原始数据的 12 倍），不适合 GB 级大文件。
- **脆弱性**：伪装文件若被某些编辑器“重新保存”，可能丢失零宽字符导致密文永久损坏。
- **无后门**：忘记密码 = 永远无法恢复（这是加密设计，但对用户是风险）。
- **单文件工具**：当前仅支持命令行，无 GUI。

## 📦 安装依赖

- Python **3.10 或更高版本**

pip install pycryptodome
```

（无需额外安装 `zlib`、`hashlib` 等，均为标准库）

## 🚀 快速使用

### 1️⃣ 伪装加密（推荐）

```bash
# 伪装文字直接输入
python GhostFile.py -w "糯米狐" 秘密笔记.md

# 从文件读取伪装内容
python GhostFile.py -w 长篇文章.txt 照片.jpg
```

加密后生成 `秘密笔记.md.enc`，用记事本打开只会看到 **糯米狐** 三个字。

### 2️⃣ 普通加密（无伪装）

```bash
python GhostFile.py 财务报表.xlsx
```

生成二进制 `.enc` 文件，体积紧凑，但不具备伪装效果。

### 3️⃣ 解密

```bash
python GhostFile.py -d 秘密笔记.md.enc
```

输入正确密码后，还原出原始文件。

## 🧰 工作流程

```
原始文件
   ↓
[zlib 压缩]
   ↓
[凯撒字节位移（模256）]
   ↓
[AES-256-CBC 加密]
   ↓
  ├── 普通模式 → 直接写出 .enc 二进制
   └── 伪装模式 → Base64 → 零宽编码 → 嵌入用户指定的可见文字 → 写出 .enc 纯文本
```

解密时自动检测文件类型（是否含零宽标记），并逆向还原。

---

## ⚠️ 重要警告

- **绝对不要用文本编辑器“另存为”伪装文件**，这几乎一定会破坏零宽字符，导致数据永久丢失！
- 移动或复制文件时，请使用文件管理器直接拷贝，不要通过文本编辑器中转。
- 密码请离线安全保存，没有任何找回机制。
